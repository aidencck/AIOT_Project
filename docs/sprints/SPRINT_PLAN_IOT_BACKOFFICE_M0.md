# AIoT 管理后台 M0 Sprint 计划与核心任务

## 1. 规划范围
- 目标版本：M0（8 周）
- Sprint 节奏：2 周 / Sprint，共 4 个 Sprint
- 管理方式：GitHub Milestone + GitHub Issues

## 2. Sprint 划分

### Sprint 1（W1-W2）：基础主数据与权限底座
**目标**
- 打通“人-家庭-设备”主数据关系，并建立后台最小权限体系。

**核心任务**
- 组织与权限中心：角色、菜单、数据权限（项目/家庭维度）。
- 用户与家庭中心：家庭创建、成员管理、设备绑定/解绑。
- 设备与资产中心：设备台账、在线状态、固件版本、心跳时间。
- 接口与数据契约：统一错误码、审计字段、缓存 Key 规范。

**验收标准**
- 用户可按权限访问对应菜单与数据范围。
- 可查询任意家庭下设备资产与最新在线状态。

### Sprint 2（W3-W4）：规则告警与通知联动
**目标**
- 异常可被规则识别，并形成可处理的告警事件。

**核心任务**
- 告警规则模板：阈值、离线、频发、去重、升级策略。
- 告警中心：告警入池、级别标注、状态流转。
- 通知中心：至少打通 1 条通知通道（短信/企业微信/邮件）。
- 告警数据结构：关联设备、家庭、规则、触发时间、严重级别。

**验收标准**
- 关键异常在规则命中后 30 秒内产生告警记录。
- 通知链路可追踪，失败可重试。

### Sprint 3（W5-W6）：工单闭环与 SLA
**目标**
- 建立“告警 -> 工单 -> 处理 -> 结案 -> 回写”的处理闭环。

**核心任务**
- 自动建单：按告警级别和类型自动创建工单。
- 工单分派：支持手动分派与规则分派。
- SLA 机制：响应时长、处理时长、超时预警。
- 结果回写：结案信息回写告警中心，沉淀处理标签。

**验收标准**
- 告警自动建单转化率 > 95%。
- P1 工单 15 分钟内响应率 > 90%。

### Sprint 4（W7-W8）：审计治理与经营看板
**目标**
- 保证全流程可追溯，并支撑 VP 日常经营决策。

**核心任务**
- 审计中心：关键操作日志、权限变更日志、异常登录审计。
- 高风险操作：二次确认与告警（如批量解绑、权限提升）。
- 经营看板：在线率、告警趋势、SLA 达成率、一次解决率。
- 发布与演练：回滚预案演练、运维手册、值班机制。

**验收标准**
- 关键操作审计覆盖率 100%。
- VP 看板日更成功率 100%。

## 3. 核心任务总览（跨 Sprint）
- 权限与主数据：组织权限、用户家庭、设备资产。
- 事件与处置：规则告警、通知联动、工单闭环。
- 治理与决策：审计风控、经营看板、发布治理。

## 4. 建议标签体系（GitHub）
- `sprint:m0-s1`、`sprint:m0-s2`、`sprint:m0-s3`、`sprint:m0-s4`
- `epic:access`、`epic:alert`、`epic:ticket`、`epic:governance`
- `priority:p0`、`priority:p1`

## 5. 远程管理规范
- 每个核心任务必须绑定 `Milestone + Label + Owner`。
- 每个 Sprint 每周至少一次状态更新（风险、阻塞、偏差）。
- 每个 Sprint 结束提交验收结论（完成度、问题清单、改进行动）。
