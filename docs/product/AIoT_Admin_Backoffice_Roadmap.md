# AIoT 管理后台产品路线图（最小闭环导向）

## 1. Premise（前提）
- 当前平台已具备设备、家庭、用户等基础能力，但缺少统一的后台运营与治理入口。
- 管理后台首要目标不是“大而全”，而是先打通 IoT 运维核心业务闭环。
- 产品成功标准：问题可发现、任务可流转、过程可追溯、结果可复盘。

## 2. Constraints（约束）
- 首期交付周期控制在 8 周，优先实现 P0 能力。
- 权限采用 RBAC + 数据范围隔离（项目/家庭级），不做复杂 ABAC。
- 看板采用固定指标，不引入自助 BI 平台。
- 不新增高复杂外部依赖，优先复用现有微服务与缓存能力。

## 3. Boundaries（边界）
- 包含：组织权限、用户家庭、设备资产、告警规则、工单流转、审计追踪、基础看板、系统配置。
- 不包含：营销自动化编排、复杂财务结算、预测性智能运维、全量自定义报表。

## 4. Endgame（终局）
- 建设“业务运营中枢 + 风险控制中枢 + 组织协作中枢”一体化后台。
- 支撑跨团队标准化运营，形成可复制的区域/项目管理模式。
- 为后续智能化（预测告警、自愈策略、策略推荐）提供高质量过程数据底座。

## 5. 最小闭环业务主链路
`设备接入` → `家庭/项目绑定` → `规则监控` → `告警触发` → `工单处理` → `结果回写` → `经营复盘`

## 6. 模块规划（按价值优先级）

### P0（必须，闭环必需）
- 组织与权限中心：账号、角色、菜单、数据权限、关键操作二次确认。
- 用户与家庭中心：用户档案、家庭档案、成员关系、设备绑定/解绑。
- 设备与资产中心：设备模型、设备台账、在线状态、固件版本、最近心跳。
- 告警与规则中心：阈值规则、离线规则、频发规则、告警去重、升级策略。
- 客服与工单中心：告警自动建单、分派处理、SLA 计时、结案回写。
- 审计与风控中心：操作日志、权限变更日志、异常登录告警。
- 数据分析中心（基础版）：在线率、告警趋势、SLA 达成率、一次解决率。
- 系统配置中心：消息模板、通知渠道、字典配置、第三方连接配置。

### P1（增长强化）
- 商品/服务内容运营（IoT 场景下的套餐、能力包、服务项管理）。
- 客服知识库与质检能力。
- 财务与结算基础对账（不含复杂分账与发票自动化）。

### P2（智能优化）
- 可视化规则编排与策略 A/B。
- 故障预测、流失预警、运营自动化。
- AIOps 自愈建议与闭环自动执行。

## 7. 角色与职责映射
- 平台管理员：组织、权限、配置、审计策略。
- 运维工程师：设备状态、规则策略、告警处理、工单协同。
- 客服主管：工单队列管理、SLA 监控、质检复盘。
- 财务对账员：异常退款/赔付/账务核对（P1 后增强）。

## 8. VP 核心管理指标
- 增长：家庭活跃率、设备活跃率、功能使用渗透率。
- 运营效率：告警到响应时长、工单平均闭环时长、人效处理量。
- 风险治理：高风险告警占比、误报率、审计覆盖率。
- 质量稳定：设备在线率、关键任务成功率、系统可用性。

## 9. 验收口径（M0）
- 告警自动建单转化率 > 95%。
- P1 告警工单 15 分钟内响应率 > 90%。
- 工单平均闭环时长较当前下降 30%+。
- 关键操作审计追溯覆盖率 = 100%。
- VP 看板日更成功率 = 100%。
