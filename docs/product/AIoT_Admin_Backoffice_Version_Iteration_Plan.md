# AIoT 管理后台版本迭代计划（M0-M2）

## 1. 版本策略
- `M0（8周）`：以最小闭环为目标，优先交付 P0 功能，形成可运营、可追溯、可复盘的后台。
- `M1（6-8周）`：增强运营与增长能力，补齐服务体验与基础财务治理。
- `M2（8-12周）`：引入智能化与自动化，提升规模化运营效率。

## 2. M0 里程碑拆解（最小闭环）

### M0-W1~W2：基础底座与数据主档
- 组织与权限中心（RBAC + 数据范围）上线。
- 用户与家庭中心上线（家庭、成员、设备关系管理）。
- 设备与资产中心上线（台账、状态、版本、心跳）。
- 交付标准：核心实体可增删改查，权限隔离可验证。

### M0-W3~W4：规则监控与告警触发
- 告警与规则中心上线（阈值、离线、频发、去重）。
- 通知能力接入（短信/企业微信/邮件至少一种通道）。
- 告警中心完成事件入池与优先级标注。
- 交付标准：关键异常可被规则捕获并生成可处理告警。

### M0-W5~W6：工单闭环与协同处理
- 客服与工单中心上线（自动建单、分派、处理、结案）。
- SLA 指标可视化（响应时长、处理时长、超时预警）。
- 处理结果支持结构化回写（故障类型、处理动作、结论）。
- 交付标准：告警到工单可闭环，处理过程全链路可追踪。

### M0-W7：治理与审计
- 审计与风控中心上线（操作日志、权限变更日志、异常登录）。
- 高风险操作二次确认（如批量解绑、权限提升）。
- 交付标准：关键行为“谁在何时做了什么”可完整追溯。

### M0-W8：经营看板与发布验收
- 数据分析中心（基础版）上线（在线率、告警量、SLA、一次解决率）。
- 发布演练、回滚演练、操作手册与培训完成。
- 交付标准：VP 可通过看板完成日常经营决策与周复盘。

## 3. M1 迭代目标（增长强化）
- 营销与服务运营模块：服务包、优惠策略、活动触达。
- 客服知识库 + 质检：提升一次解决率与交付一致性。
- 财务对账基础能力：异常赔付与账务核对流程化。
- 阶段指标：复购/续费提升、客服人效提升、异常账务下降。

## 4. M2 迭代目标（智能优化）
- 规则平台升级：策略编排、版本管理、灰度发布。
- 智能分析：故障预测、工单量预测、风险预警。
- 自动化运营：策略推荐与半自动执行。
- 阶段指标：人工介入率下降、预测准确率提升、处理时效进一步下降。

## 5. 版本发布与治理机制
- 发布节奏：双周小版本、月度里程碑版本。
- 需求入口：统一进入需求池，按“业务价值/实施成本/风险等级”评分排序。
- 评审机制：产品评审 + 技术评审 + 上线评审三道关。
- 质量门禁：核心链路测试通过、回滚预案验证、审计日志可用。

## 6. Jira 映射建议（用于远程协作）
- Epic-1：组织权限与用户家庭主数据。
- Epic-2：设备资产与状态治理。
- Epic-3：规则告警与通知中心。
- Epic-4：工单协同与 SLA 管理。
- Epic-5：审计风控与合规治理。
- Epic-6：经营看板与版本发布治理。

## 7. 风险与缓释
- 规则误报高：先上线保守阈值 + 人工校准窗口。
- 工单流转拥堵：引入优先级队列和自动分派策略。
- 数据口径不一致：统一指标字典并固化 ETL 口径。
- 权限边界不清：以“最小权限原则”默认收敛授权。
