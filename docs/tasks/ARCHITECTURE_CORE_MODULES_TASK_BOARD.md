# AIOT 项目核心模块与核心链路任务板

## 1. 背景与目标

- 目标：将现有“架构梳理结论”沉淀为可执行任务，便于按里程碑推进和追踪。
- 范围：聚焦当前已落地主链路（Gateway/Auth/Device/Home）和在建链路（Rule Engine 事件执行）。
- 原则：先保稳定与可观测，再做能力扩展；先补齐链路闭环，再做智能化深化。

## 2. 架构基线（当前）

- 已落地核心模块：`aiot-gateway`、`aiot-auth-service`、`aiot-device-service`、`aiot-home-service`、`aiot-common`。
- 在建模块：`aiot-rule-engine`、`aiot-mqtt-adapter`、`aiot-data-parser`、`aiot-shadow-service`。
- 基础依赖：`Nacos + MySQL + Redis + EMQX`，以 `Docker Compose` 单节点部署为主。

## 3. 任务分解（按核心链路）

### A. 设备接入与状态链路（P0）

- [x] **A1** 梳理并固化 EMQX 鉴权与 Webhook 配置模板（鉴权、签名、时间窗、防重放）
  - 交付物：统一配置说明、环境变量清单、验收脚本
  - 验收标准：设备上线/下线状态 100% 可回写设备域，异常请求被拒绝
- [x] **A2** 补齐状态同步失败的可观测性与重试策略治理
  - 交付物：失败分类日志、重试策略说明、告警触发条件
  - 验收标准：同步失败可定位（原因可分层），出现连续失败可告警
- [x] **A3** 统一设备状态机与状态流转约束文档
  - 交付物：状态流转矩阵（未激活/在线/离线）、错误码与提示约定
  - 验收标准：非法状态流转全部被拦截且错误信息可读

### B. 家庭权限与跨服务补偿链路（P0）

- [x] **B1** 统一 Home->Device 权限检查契约与超时重试参数
  - 交付物：调用契约文档、参数基线（超时/重试次数/退避策略）
  - 验收标准：设备域所有家庭资源操作均经权限校验
- [x] **B2** 强化家庭/房间删除补偿的幂等性与审计记录
  - 交付物：补偿调用幂等约定、审计字段规范、失败处理流程
  - 验收标准：重复触发补偿不会产生脏数据，补偿日志可追溯
- [x] **B3** 内部接口安全基线巡检（`/api/v1/internal/**`）
  - 交付物：接口清单、鉴权检查结果、风险项与修复计划
  - 验收标准：内部接口全部强制服务令牌校验

### C. 设备配网与影子链路（P1）

- [x] **C1** 配网 token 生命周期治理（TTL、一次性消费、异常审计）
  - 交付物：token 生命周期说明、并发与重放用例
  - 验收标准：token 仅可成功兑换一次，过期/篡改请求均拒绝
- [x] **C2** 设备影子字段模型约束与版本策略
  - 交付物：`desired/reported` 字段约束、冲突合并策略
  - 验收标准：影子写入具备基本校验，读取结构稳定

### D. 规则引擎事件链路（P1）

- [x] **D1** 将规则存储从内存态升级为持久化（最小可用）
  - 交付物：规则表结构、读写接口、迁移说明
  - 验收标准：服务重启后规则不丢失，可继续执行
- [x] **D2** 规则执行动作从日志扩展到可配置动作
  - 交付物：动作类型定义（如告警通知、Webhook）、参数校验规范
  - 验收标准：至少 1 类真实动作可用并可回溯执行结果
- [x] **D3** 事件处理可靠性治理（幂等键、重复消费防护）
  - 交付物：事件幂等策略、去重键规范、异常回放说明
  - 验收标准：同一事件重复投递不导致重复动作副作用

## 4. 里程碑规划

- **M0（1\~2 周）**：完成 A+B（P0）并上线最小可观测能力
- **M1（2\~4 周）**：完成 C+D（P1）并形成可持续迭代基线

## 5. 风险与依赖

- 风险：内部令牌配置不一致导致跨服务调用失败。
- 风险：规则引擎仍为骨架，快速扩展时易引入状态一致性问题。
- 依赖：Nacos/Redis/MySQL/EMQX 的环境一致性；CI 增量构建策略稳定可用。

## 6. 看板使用方式

- 每周例会更新：任务状态、阻塞项、下周目标。
- 状态建议：`TODO / IN_PROGRESS / BLOCKED / DONE`。
- 追踪维度：链路成功率、错误率、平均恢复时长（MTTR）、回归缺陷数。
